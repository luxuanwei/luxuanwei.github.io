Dockerfile入门代码

发表于 2016-03-16

FROM

用于拉取基础镜像

FROM <image> 或 FROM <image>:<tag>

MAINTAINER

指定维护者信息

MAINTAINER <name>

RUN

在当前镜像基础上执行命令，”\”用于换行

RUN <shell command>
RUN ["executable","param1","param2"]

CMD

指定启动容器时执行的命令

如果在容器中指定了多条指令，只执行最后一条

CMD ["executable","param1","param2"] 使用exec执行
CMD [command param1 param2 ] 在/bin/sh 中执行
CMD ["param1","param2"]

EXPOSE

告诉Docker服务端暴露的端口号

在启动时指定容器主机，并自动分配一个端口

转发到指定的端口

ENV

指定一个环境变量，会被后续RUN指令使用，并在容器运行时保持

ENV PATH /usr/local/postgres-$PG_MAJOR/bin:$PATH

ADD

复制指定的< src >到容器中的< dest >

< src >可以是Dockerfile所在目录的一个路径，也可以是一个URL或tar文件

COPY

复制本地主机的< src >到容器中的< dest >

ENTRYPOINT

配置容器启动后执行的命令，并且不可被docker run提供的参数覆盖

每个Dockerfile只能有一个 ENTRYPOINT

ENTRYPOINT ["executable","param1","param2"]
ENTRYPOINT command param1 param2

VOLUME

创建一个可以从本地主机护着其他容器挂载的挂载点，一般用来存放数据库

VOLUME ["/data"]

USER

指定运行容器时的用户名或UID，后续的RUN也会使用指定用户

USER daemon

WORKDIR

为后续的RUN,CMD,ENTRYPOINT指令配置工作目录

ONBUILD

配置当前创建镜像作为其他新镜像的基础镜像

ONBUILD [INSTRUCTION]

使用github工具直接在蜂巢上部署2048

发表于 2016-03-15

简介

此文档主要演示以2048小游戏为例，如何使用github工具直接在蜂巢上部署各种网页。

准备条件

1.本地创建Dockerfile和网页文档（war格式），这里给出tomcat的dockerfile

FROM ubuntu:latest

ENV REFRESHED_AT 2015-7-28
RUN apt-get -yqq update
RUN apt-get -yqq install tomcat7 default-jdk

ENV CATALINA_HOME /usr/share/tomcat7
ENV CATALINA_BASE /var/lib/tomcat7
ENV CATALINA_PID  /var/run/tomcat7.pid
ENV CATALINA_SH   /usr/share/tomcat7/bin/catalina.sh
ENV CATALINA_TMPDIR /tmp/tomcat7-tomcat7-tmp
COPY ./ /var/lib/tomcat7/webapps

RUN mkdir -p $CATALINA_TMPDIR
VOLUME ["/var/lib/tomcat7/webapps/"]
EXPOSE 8080 9000
ENTRYPOINT ["/usr/share/tomcat7/bin/catalina.sh", "run" ]

2.将相应代码上传至github

快速上手

点击“创建服务”按钮，进入服务创建界面。

选择任意一个官方镜像（不需要是php），在Git地址处输入对应的github地址

这里提供一个github地址

https://github.com/luxuanwei/2048.git

在Dockerfile处选择自定义，输入Dockerfile文件的源码中的路径。

注：请将目的端口更改为8080

结果

经过短暂的日志进度提示，容器很快创建成功，就可以直接看到容器挂载的外网IP

现在直接在浏览器上打开界面上的地址，就可以快速访问刚成功创建的容器启动的服务

http://公网ip/2048

通过这样的方法，我们的dockerfile就可以在不通过本地运行上传的方式直接在蜂巢上面运行了。

通过github和hexo布局博客

发表于 2016-03-15

简介

此文档主要演示如何通过github和hexo进行博客的部署。

准备条件

本地安装好github工具和nodejs工具，，可以点击官网直接下载安装。
准备一个github账号，并建立一个命名为 username.github.io 的仓库

打开命令行，安装hexo

npm install -g hexo
npm install hexo-server
npm install hexo-deployer-git --save

快速上手

在D盘建立hexo文件夹，用git bash打开

输入命令

hexo init 
hexo new “first passage”
hexo g
hexo s

更改 _config.yml

光标定位到最后三行，对以下配置进行更改

deploy：
 type: git
 repository: http://github.com/username/username.github.io.git
 branch: master'

ps:冒号后面需要空格，否则会无法读取，

hexo deploy

更新博客代码

hexo clean
hexo g
hexo s

SSH密钥设置

首先我们需要检查你电脑上现有的ssh key：

‘ $ cd ~/. ssh 检查本机的ssh密钥 ‘

生成新的SSH Key：

‘$ ssh-keygen -t rsa -C 邮件地址@youremail.com’
