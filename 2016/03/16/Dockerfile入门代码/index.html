Dockerfile入门代码

FROM

用于拉取基础镜像

FROM <image> 或 FROM <image>:<tag>

MAINTAINER

指定维护者信息

MAINTAINER <name>

RUN

在当前镜像基础上执行命令，”\”用于换行

RUN <shell command>
RUN ["executable","param1","param2"]

CMD

指定启动容器时执行的命令

如果在容器中指定了多条指令，只执行最后一条

CMD ["executable","param1","param2"] 使用exec执行
CMD [command param1 param2 ] 在/bin/sh 中执行
CMD ["param1","param2"]

EXPOSE

告诉Docker服务端暴露的端口号

在启动时指定容器主机，并自动分配一个端口

转发到指定的端口

ENV

指定一个环境变量，会被后续RUN指令使用，并在容器运行时保持

ENV PATH /usr/local/postgres-$PG_MAJOR/bin:$PATH

ADD

复制指定的< src >到容器中的< dest >

< src >可以是Dockerfile所在目录的一个路径，也可以是一个URL或tar文件

COPY

复制本地主机的< src >到容器中的< dest >

ENTRYPOINT

配置容器启动后执行的命令，并且不可被docker run提供的参数覆盖

每个Dockerfile只能有一个 ENTRYPOINT

ENTRYPOINT ["executable","param1","param2"]
ENTRYPOINT command param1 param2

VOLUME

创建一个可以从本地主机护着其他容器挂载的挂载点，一般用来存放数据库

VOLUME ["/data"]

USER

指定运行容器时的用户名或UID，后续的RUN也会使用指定用户

USER daemon

WORKDIR

为后续的RUN,CMD,ENTRYPOINT指令配置工作目录

ONBUILD

配置当前创建镜像作为其他新镜像的基础镜像

ONBUILD [INSTRUCTION]
